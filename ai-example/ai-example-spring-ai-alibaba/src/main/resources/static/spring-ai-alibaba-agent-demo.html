Spring AI Alibaba ReactAgent 联调控制台

这个页面专门服务于 ai-example-spring-ai-alibaba 模块，适合一边看文档一边验证 simple_agent、order_assistant 和 order_summary_agent 的实际调用效果。 你可以在同一页里直接测试同步问答、流式输出、结构化 JSON 以及会话记忆状态。

模块：ai-example-spring-ai-alibaba
端口：7089
模型：DeepSeek · deepseek-chat
后端地址：http://localhost:7089
重点：Tools / Memory / Hook / Interceptor / OutputType

Agent APIs

适合怎么讲解

Teaching Flow
  1. 先点击“运行 simple_agent”，验证最小 Agent 只走模型调用。
  2. 再用“订单同步对话”和“订单流式输出”观察工具调用与会话记忆。
  3. 最后用“结构化总结”和“查看线程状态”串起 outputType 与 threadId。

讲解提示

Live Notes
页面已准备好。推荐顺序：先用 simple_agent，再测试 chat / stream，最后看 summary 和 thread-state。

调试建议

配合文档一起演示更直观
  1. 如果流式窗口一块块连续出现文本，说明浏览器已经把 text/event-stream 正常消费了。
  2. 连续用同一个 sessionId 提问两次，再点“查看线程状态”，可以直接看到 messageCount 增长。
  3. 切换不同 sessionId，等于切换不同线程，会话历史不会串台。
